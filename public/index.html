// PROCESSING PIPELINE

7-Step Intake Pipeline

From inbound call to qualified lead notification in under 30 seconds

01
Validate Payload
Check that name, phone, and case type are present. Missing data gets logged to Error Log and admin is alerted.
02
Dedup Check (4 Layers)
Call ID match, phone + 24hr window, webhook dedup, and Sheets safety net. Duplicates update existing records.
03
AI Enrichment
OpenAI GPT-4o with function calling extracts structured data. Enum fields enforce consistency, required fields guarantee minimum data.
04
Two-Layer Qualification
Layer 1: Deterministic rules for obvious cases (hard qualifiers/disqualifiers). Layer 2: AI judgment for gray areas. Always errs on the side of qualifying.
05
Store Lead Data
Parallel writes to CRM (Leads sheet) and audit trail (Lead Log sheet). Exponential backoff retry: 5s, 30s, 2min.
06
Route Notifications
Qualified: email intake team + calendar event. Not qualified: polite decline with State Bar referral. Urgent cases get fast-tracked.
07
Clean Up & Log
Update Dedup Tracker, mark Lead Log as complete. All operations logged for full audit trail.
// QUALIFICATION ENGINE

Core Logic

Two-layer qualification: deterministic rules that can't hallucinate, plus AI judgment for nuanced cases

qualification-logic.js
function qualifyLead(data) { // HARD DISQUALIFIERS — override AI, no exceptions if (!FIRM_CONFIG.practiceAreas.includes(data.case_type)) return { qualified: false, reason: 'Outside practice areas' }; if (data.existing_representation === true) return { qualified: false, reason: 'Already represented' }; // HARD QUALIFIERS — catch leads the AI might undervalue if (data.urgency === 'high') return { qualified: true, reason: 'Urgent case in practice area' }; if (data.case_type === 'personal_injury' && data.injuries) return { qualified: true, reason: 'PI with reported injuries' }; // DEFAULT: Trust AI's judgment for gray areas return { qualified: data.qualified, reason: data.qualification_reason }; }
